The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Field Control
  • Password
  • Encryption

The PASSWORD component allows to encrypt the user's input. The encryption transfers the password into a hashed string using diverse one-way encryption methods.

Pay attention: this type of encryption just makes sure that the user's password input (e.g. "aaaa") is immediatly encrypted as part of the componnt's processing (e.g. into "sb437fdbs7fsdf=="). The encrypted value is the one that is sent to server side. There is no way (of course...) to decrypt back into "aaaa", so on server side you from now on have to "live" with the encrypted password.

And, pay attention, too: this encryption does only hide the actual value that the user enters. I.e. no one can see (e.g. by listening to the http protocol) that the user enters "aaaa". But: esp. when using http, then the encryption of the PASSWORD component does not add any kind of security to your application! - Still someone can listen to what is transferred via http, and then use the encrypted password to the server side!

Using https is the only way to really make sure that password values are managed in a secure way!
