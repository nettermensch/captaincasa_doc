The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Text Input/Output
  • Text input with AI support
  • Text creation by key words

This component shows how to use AI functions for creating texts in some "guided text editor".

Behind the scenes the component talks to ChataGPT which is abstracted behind some interface "IGuidedTextService".

Each text type declares some variables to be filled up front. The variables are arranged in some start prompt which is sent to create the text.

Further one the text can be modified either free style - or by pre-configured prompts.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:beanprocessing id="g_1" beanbinding="#{d.DemoGuidedTexteditor}" />
<t:rowdemobodypane id="g_4" objectbinding="#{d.DemoGuidedTexteditor}" >
<t:row id="g_6" >
<tpbc:guidedtexteditor id="g_2" shownullcontent="true" text="#{d.DemoGuidedTexteditor.text}" />
</t:row>
</t:rowdemobodypane>
</t:layoutdefinition>
