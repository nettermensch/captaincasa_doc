The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • General Issues
  • Error Management

The demo demonstrates how errors (which should never happen of course... ;-) ...) are managed on server and on client side.

The one category of errors is, that something on the way from the client to the server and back happened: this can reach from connection failures to errors on server side. In the example the server causes an Error that is thrown.

The second category of errors is, that something happens on client side. This should only happen if the XML layout definition that is received by the client is not properly formed. (Note: You can build up the page's XML dynamically...!)

In both cases it is essential to have a look onto the server and the client log that is written.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.demoError}" rowdistance="#{ccstylevalue['@defaultrowdistance@']}" >
<t:row id="g_2" >
<t:label id="g_3" text="First Name" width="100" />
<t:field id="g_4" flush="true" text="#{d.DemoError.firstName}" width="100" />
</t:row>
<t:row id="g_5" >
<t:coldistance id="g_6" width="100" />
<t:textpane id="g_7" font="size:10" text="When defining value &#34;error&#34; then an error is thrown within the corresponding set-method on server side." width="100%" />
</t:row>
<t:rowdistance id="g_8" height="2" />
<t:row id="g_9" >
<t:button id="g_10" actionListener="#{d.demoError.onThrowError}" text="Throw error on server side" width="100%" />
</t:row>
<t:row id="g_11" >
<t:button id="g_12" actionListener="#{d.demoError.onThrowSevereError}" text="Throw severe error on server side" width="100%" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;
import javax.faces.event.ActionEvent;

import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.elements.ApplicationErrorInfo;
import org.eclnt.jsfserver.elements.ApplicationErrorInfoDuringSet;
import org.eclnt.jsfserver.elements.impl.FIELDComponentTag;
import org.eclnt.jsfserver.elements.impl.PANEComponent;
import org.eclnt.jsfserver.util.ClientTestLog;
import org.eclnt.jsfserver.util.IClientLogAware;
import org.eclnt.jsfserver.util.IErrorAware;
import org.eclnt.util.log.CLog;
import org.eclnt.workplace.IWorkpageDispatcher;



public class DemoError 
    extends WorkpageDispatchedPageBean
    implements IErrorAware
{
    PANEComponent m_pane;
    String m_firstName;
    
    public DemoError(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
    }
    
    public void setFirstName(String value) 
    { 
        m_firstName = value; 
        if ("error".equalsIgnoreCase(value) || "severeerror".equalsIgnoreCase(value))
        {
            ClientTestLog.addClientTestLogMessage("throwing severe error during set");
            throw new Error("What a strange value!");
        }
    }
    public String getFirstName() { return m_firstName; }
    
    public PANEComponent getPane() { return m_pane; }
    public void setPane(PANEComponent pane) { m_pane = pane; }
    
    public void onThrowError(ActionEvent ae)
    {
        ClientTestLog.addClientTestLogMessage("throwing error");
        throw new Error("This error was thrown on server side.");
    }
    
    public void onThrowSevereError(ActionEvent ae)
    {
        ClientTestLog.addClientTestLogMessage("throwing severe error");
        throw new Error("SEVERE ERROR");
    }
    
    // Please NOTE: in the demo workplace there is also a second level of catching errors - on
    // dispatcher level. Because top-levels override the lower levels, there are the same 
    // checks for SEVERE ERRPRs within the Dispatcher.java-class
    
    public void onApplicationErrorDuringSet(ApplicationErrorInfoDuringSet aeids)
    {
        ClientTestLog.addClientTestLogMessage("managing error during set");
        Statusbar.outputError("Error occurred on server side during setting a property value: " + aeids.getException());
        if ("severeerror".equals(m_firstName))
            throw new Error("Sever error is too much...!");
    }
    
    public void onApplicationError(ApplicationErrorInfo aei)
    {
        ClientTestLog.addClientTestLogMessage("managing severe error");
        Throwable t = aei.getException();
        if (t.getMessage().indexOf("SEVERE ERROR") >= 0)
            throw new Error(t);
        Statusbar.outputError("Error occurred on server side during invocation of method: " + t);
    }
    
}
