The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Field Control
  • Auto Completion
  • Example

The AUTOCOMPLETE component allows the user to input some text. After a certain time of inactivity a URL request is executed in order to pick value-proposals from the server. In the case of AUTOCOMPLETE this request is sent completely decoupled from the normal request/response processing of the Enterprise Client.

The AUTCOMPLETE component expects a certain URL, which you should normally provide by creating a corresponding "AutoCompleteProvider" implementation on backend side.
