The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Field Control
  • Field Value Help

The COMBOFIELD component adds a valid value help to the normal FIELD component. This means: the component talks back to the server side actionListener when the user either presses the F4 key or clicks onto the corresponding icon within the component.

On server side you are in principal open "to do what you want". Typically you will use a popup in order to provide a selection of values.

For simple value lists, consisting out of either a list of ids or a list of ids with their associated text, there is a default handling, using class "IdTextSelection". Please have a look into the code.

In the "department example" just the selected id is passed back into the server processing. In the "cost center example" both the id and text are passed back.

As shown in the example with "Name": by combining the properties "FLUSH" and "FLUSHTIMER" you can easily build up complex scenarios, in which a value proposal is done based on the user's current input - and without the user explicitly requesting the value help, but controlled by a timer that is activated when the user does not do any input into the field.

There is an additional interesting attribute: KEEPFOCUS. In this case the focus is not moved into the value help popup but is kept within the COMBOFIELD component. Ther user can continue typing. By pressing cursor-down the user can navigate into the popup. Of course this mode only is working correctly if the server side processing opens up a modeless window.

        }

            m_modelessPopup.setCloseonclickoutside(true);
            m_modelessPopup.setStartfromrootwindow(false);
        }
