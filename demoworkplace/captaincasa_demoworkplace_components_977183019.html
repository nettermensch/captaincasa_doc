The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Text Input/Output
  • Text with Links

The TEXTWITHLINKS component renders an HTML text containing links.

When the user clicks onto a link then a server side action is triggered (ActionEvent of type BaseActionEventLink). The event contains the information about the href-attribute that you define with the link definition. The href-attribute may not be a valid URL as consequence, but just some "link-id" that you then process on server side.

Example: when clicking onto a link with <a href='AAA'>Hallo</a>, then on server side the action listener is invoked, the event of type BaseActionEventLink can be checked with "getLinkId()", returning "AAA".

In the demo there are two ways shown how to open http-links:
(.) The first one is the "through the server" one. The link's href-definition is sent to the server side - just as any other link. The server side then triggers a JSHOWURL component.
(.) The seconds one is the "through the client" one: by adding prefix "url:" to the href-definition, the client immediately knows what to do - and opens up the URL in a new window. By default target "_blank" is used, you may set your own target by append ";target:..." to the href-definition.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:beanprocessing id="g_1" >
<t:jshowurl id="g_2" trigger="#{d.DemoTextWithLinks.browserTrigger}" url="#{d.DemoTextWithLinks.browserURL}" useappletcontext="false" usedesktop="true" usejavascript="false" />
</t:beanprocessing>
<t:rowdemobodypane id="g_3" objectbinding="#{d.DemoTextWithLinks}" >
<t:row id="g_4" >
<t:textwithlinks id="g_5" actionListener="#{d.DemoTextWithLinks.onLinkAction}" border="top:1;bottom:1;left:1;right:1;color:#00000030;top2:5;bottom2:5;left2:5;right2:5;color2:#00000000" font="family:Tahoma" htmlstylesheetrule="A {color:#000000}" text="#{d.DemoTextWithLinks.html}" width="100%" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.elements.events.BaseActionEventLink;
import org.eclnt.jsfserver.elements.util.Trigger;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoTextWithLinks}")

public class DemoTextWithLinks
    extends PageBean
    implements Serializable
{
    protected String m_browserURL;
    public String getBrowserURL() { return m_browserURL; }
    public void setBrowserURL(String value) { this.m_browserURL = value; }

    protected Trigger m_browserTrigger = new Trigger();
    public Trigger getBrowserTrigger() { return m_browserTrigger; }
    public void setBrowserTrigger(Trigger value) { this.m_browserTrigger = value; }

    public DemoTextWithLinks(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("textwithlinks");
    }
    

    protected String m_html = 
        "<html>" +
        "<body>" +
        "This text contains some links: <a href='AAA'>First link (AAA)</a> and <a href='BBB'> Second link (BBB)</a>. When clicking a corresponding information will be sent to the server side - so that your application can react correspondingly. " +
        "<br><br>" +
        "This text contains some links: <a href=\"AAA\">First link (AAA)</a> and <a href=\"BBB\"> Second link (BBB)</a>. When clicking a corresponding information will be sent to the server side - so that your application can react correspondingly. " +
        "<br><br>" +
        "This text contains some hyper links: <a href='http://www.google.de' target='_blank'>Google</a> and <a href='http://www.captaincasa.com'>CaptainCasa</a>. When clicking a corresponding information will be sent to the server side - so that your application can react correspondingly. " +
        "<br><br>" +
        "You also may directly open links by using prefix &quot;url:&quot;: <a href='url:http://www.google.de'>Google</a> "
        + "and <a href='url:http://www.captaincasa.com;target:CC'>CaptainCasa</a>. "
        + "In this case the navigation is completely managed on client side - without going through the server. "
        + "You may also append a target by adding &quot;;target:...&quot; to the URL." +
        "</body>" +
        "</html>";
    public String getHtml() { return m_html; }
    
    public void onLinkAction(ActionEvent event)
    {
        if (event instanceof BaseActionEventLink)
        {
            BaseActionEventLink bael = (BaseActionEventLink)event;
            Statusbar.outputSuccess("Link with id '"+bael.getLinkId()+"' was pressed!");
            if (bael.getLinkId().startsWith("http://"))
            {
                m_browserURL = bael.getLinkId();
                m_browserTrigger.trigger();
            }
        }
    }
}
