The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Scheduling
  • Padding definition

You can define a padding on SCHEDULE level that defines the distance that is kept to contained SCHEDULEITEM instances.

You may either define one value, which is applied to the top and the bottom, or you can explicitly define the top and the bottom padding.

In case the SCHEDULE is oriented vertically (attribute ORIENTATION) then you may use the left and the right padding for assigning two different padding values.
