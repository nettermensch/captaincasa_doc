The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Check Box & Radio Button
  • Check Box

The CHECKBOX component maintains a boolean value. On server side it either can be implemented as "boolean" type property or as "Boolean" object property.

When implementing as object you can take use of the TRIPLESTATE attribute. In this case the check box is able to render three states: selected (true), de-selected (false) and "partially selected" (null).

When do you need a triple state checkbox? Typically when you position the checkbox into a tree. "Partially selected" then tells the user that some tree nodes below one tree node are selected, but not all.

This is the Java code:

package workplace;

import javax.faces.event.ActionEvent;

import org.eclnt.workplace.IWorkpageDispatcher;

public class DemoCheckbox extends PageBean
{
    boolean m_value;
    boolean m_value2;
    Boolean m_tripleValue;

    public DemoCheckbox(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("checkbox");
    }
    
    public boolean isValue()
    {
        return m_value;
    }
    public void setValue(boolean value)
    {
        m_value = value;
    }
    
    
    public boolean isValue2()
    {
        return m_value2;
    }
    public void setValue2(boolean value2)
    {
        m_value2 = value2;
    }
    
    public Boolean getTripleValue()
    {
        return m_tripleValue;
    }
    public void setTripleValue(Boolean value)
    {
        m_tripleValue = value;
    }
    
    public void onSetTrue(ActionEvent ae)
    {
        m_tripleValue = new Boolean(true);
    }
    
    public void onSetFalse(ActionEvent ae)
    {
        m_tripleValue = new Boolean(false);
    }
    public void onSetBetween(ActionEvent ae)
    {
        m_tripleValue = null;
    }
}
