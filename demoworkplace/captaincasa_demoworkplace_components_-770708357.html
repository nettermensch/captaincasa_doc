The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Images
  • SVG images - coloring and sizing

Flat SVG icons can be accessed via some special URL, in which you can pass the color and the size of the icon. Is is possible as consequence to use one SVG definition for multiple colors and sizes.

The format is: /[DIR.]NAME.COLOR.WIDTHxHEIGHT.ccsvg

Example: if the SVG-file in the webcontent is "/images/svgicons/icon.svg" then a valid URL would be: "/images.svgicons.icon.#800000.32x32.ccsvg"

On server side the file either may be located in the webcontent directory, or it may be located within the classes that are loaded.

In addition within an image you may directly access variables from CaptainCasa style definitions - so that you can outsource color and style values into the style. This is a very strong feature because the color of flat icons typically depends on the background styling of the area that it is located on.

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedPageBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoSVGImageColorSize}")

public class DemoSVGImageColorSize
    extends PageBean 
    implements Serializable
{
    public DemoSVGImageColorSize(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demosvgimagecolorsize.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoSVGImageColorSize}"; }
}
