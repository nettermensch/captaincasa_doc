The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Grids & Trees
  • Grids and Trees
  • Horizontal Scrolling

The horizontal scroll bar of the grid can be explicitly controlled by using the attribute HORIZONTALSCROLLMODE. You can test the behaviour of the grid for each mode within this demo by resizing the contained column.

There are four modes:

"hidden" - OK, there is obviously no horizontal scrollbar shown any time. The grid is cut, the user does not have any possibility to reach the content. This mode should be handled with care... ;-)

"auto" - This is the default mode. In order to avoid screen flickering and resizing the total height of the grid is calculated always as total of all row heights + a potential horizontal scrollbar at the bottom. If the grid does not require a horizontal scrollbar then its height is distributed equally to all other rows. You will see that there is a row height resizing when increasing the width of the corresponding column.

"autowithresize" - Now, with this mode the row height of the rows really represents the ROWHEIGHT definition of the grid. If a scrollbar is required then the total grid is resized, i.e. it notfies the whole page that it requires more vertical space, and as consequence triggers a resizing of the whole page. This mode does only make sense to be used if NOT explicitly specifying a height to the grid.

"always" - Well, now the horizontal scrollbar is always rendered.

As mentioned: the default is "auto", because it's the most performant way. You might typically use "autowithresize" when using grids without height definition and with only a few rows. "Few rows" also means, that you might use the attribute SHOWEMPTYROWS and set it to false.

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.elements.impl.FIXGRIDItem;
import org.eclnt.jsfserver.elements.impl.FIXGRIDListBinding;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoGridScrollMode}")

public class DemoGridScrollMode
    extends PageBean
    implements Serializable
{
    public class GridItem extends FIXGRIDItem implements java.io.Serializable
    {
        protected String m_name;
        public String getName() { return m_name; }
        public void setName(String value) { m_name = value; }
    }
    
    protected FIXGRIDListBinding<GridItem> m_grid1 = new FIXGRIDListBinding<GridItem>();
    protected FIXGRIDListBinding<GridItem> m_grid2 = new FIXGRIDListBinding<GridItem>();
    protected FIXGRIDListBinding<GridItem> m_grid3 = new FIXGRIDListBinding<GridItem>();
    protected FIXGRIDListBinding<GridItem> m_grid4 = new FIXGRIDListBinding<GridItem>();

    public DemoGridScrollMode(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        for (int i=0; i<100; i++)
        {
            GridItem gi;
            gi = new GridItem(); gi.setName("Name " + i); m_grid1.getItems().add(gi);
            gi = new GridItem(); gi.setName("Name " + i); m_grid2.getItems().add(gi);
            gi = new GridItem(); gi.setName("Name " + i); m_grid3.getItems().add(gi);
            gi = new GridItem(); gi.setName("Name " + i); m_grid4.getItems().add(gi);
        }
    }

    public FIXGRIDListBinding<GridItem> getGrid1() { return m_grid1; }
    public FIXGRIDListBinding<GridItem> getGrid2() { return m_grid2; }
    public FIXGRIDListBinding<GridItem> getGrid3() { return m_grid3; }
    public FIXGRIDListBinding<GridItem> getGrid4() { return m_grid3; }
}
