The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Invokers
  • Button - Click/DoubleClick

By default the button reacts on single click events - using the left mouse button.
Double-clicks are by default not transferred to the server in order to not invoke a server side action twice by accident.

You can allow double- and multiple clicks by defining the attribute DOUBLECLICKENABLED as "true".

In addition you can fine-granularly control the "quality of mouse clicks" by using the INVOKEEVENT attribute.

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.elements.events.BaseActionEventInvoke;
import org.eclnt.jsfserver.util.ClientTestLog;
import org.eclnt.jsfserver.util.IClientLogAware;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoButtonClick}")

public class DemoButtonClick
    extends WorkpageDispatchedPageBean 
    implements Serializable, IClientLogAware
{
    int m_counter = 0;
    
    public DemoButtonClick(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
        initRefComponent("button"); 
    }

    public void onPressMe(ActionEvent event) 
    {
        if (event instanceof BaseActionEventInvoke)
        {
            BaseActionEventInvoke e = (BaseActionEventInvoke)event;
            m_counter++;
            Statusbar.outputSuccess("Button was pressed ("+m_counter+"). Click count: " + e.getClickCount());
        }
    }

    public boolean onCreateClientTestLogMessage()
    {
        ClientTestLog.addClientTestLogMessage("Counter = " + m_counter);
        return true;
    }


}
