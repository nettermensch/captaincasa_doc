The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Menus
  • Button Popup - Positioning aside...

You can explicitly pass some style to the popup dialog that is opened by the BUTTONPOPUP component.

There are pre-defined style variants doing this for you. Both BUTTONPOPUP components of the example use the style to open a transparent dialog. In the dialog content there is some space on the left and on the right - so that the popup content does not hide the original button/icon that opens the dialog.

We now have the definition:
1. popup opens to the left or the right (MENUPOSITION)
2. popup automatically opens on mouse over (ACTIVATEONMOUSEOVER)
3. popup automatically closes on mouse out (CLOSONMOUSEEXIT)
4. overlapping but invisible content to show the originating button/icon

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoButtonPopupSpecialLeft}")

public class DemoButtonPopupSpecialLeft
    extends PageBean 
    implements Serializable
{
    public DemoButtonPopupSpecialLeft(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demobuttonpopupspecialleft.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoButtonPopupSpecialLeft}"; }
}
