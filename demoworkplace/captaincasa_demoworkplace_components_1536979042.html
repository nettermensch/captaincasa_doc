The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Field Control
  • Masked input
  • Masked input - value mode

If using the attribute INPUTMASK then by default the value that is passed into the server side processing is exactly the value that the user sees in the field - containing the mask character.

By using attribute INPUTMASKVALUEMODE and setting it to "withoutMask" you only receive the "net value" - without any mask character. The value is trimmed at its right side.
