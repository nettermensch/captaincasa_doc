The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Invokers
  • Hover resizing effects

The BUTTON and the PANE component provide a set of pre configured style variants that create a hover effect to resize the component. This hove effect is added into addition to coloring effects of the background.

The information about the hover effect is kept in the style. Example:

<class
n="riscbutton_withhoverresizemedium"
extends="riscbutton">
<risc
n="_hoverEffectResizeInsets"
v="5 5 5 5"/>
</class>

The CaptainCasa style definitions for the BUTTON and PANE component is made to server common use cases. You may set up the hover effects with all other components as well.

From the demo you see that the resizing effects automatically influence the sizing of the component, so that the space for resizing if automatically considered.

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoButtonHoverEffect}")

public class DemoButtonHoverEffect
    extends DemoPageBean 
    implements Serializable
{
    public DemoButtonHoverEffect(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demobuttonhovereffect.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoButtonHoverEffect}"; }
}
