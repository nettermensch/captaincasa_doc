The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Images
  • Scale image by user

You can zoom the image by...

...on desktop: use ctrl-MouseWheel
...on tablet/phone: use two finger gesture

Take a look into the layout definition. The zooming is done by a general-purpose SCALEPANE into which the actual IMAGE is embedded.

This is the XML layout definition:

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedPageBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoScaleImage}")

public class DemoScaleImage
    extends PageBean 
    implements Serializable
{
    public DemoScaleImage(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demoscaleimage.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoScaleImage}"; }
    
    float m_scale = 1f;

    public float getScale() { return m_scale; } 
    public void setScale(float scale) { m_scale = scale; }
}
