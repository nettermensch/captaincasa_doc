The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Images
  • Animated images

You can pass animated images into the client side image processing as well.
But: animated images need to indicate the animation by having "_animated" within their file name.

Background: the user can scale the client sizing using the client configuration popup (icon on the bottom right). This scaling also has impact on the size of images. For scaling images a certain API is used on client side that allows to scale up the size of images with a very good quality. This takes time but is only done once per image.
The API for scaling up images does not support animated images - so the client uses a different way of scaling up images, which is from result point of view "rougher". In order to know when to apply which scaling-algorithm you need to set the filename accordingly.
