The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Grid layout pane
  • One "leader" component drives width

In this example the fields with the red background drive the width sizing of the columns.

The corresponding fields are set to a certain width (e.g. "150"), the other fields are set to "1%". As result, due to the percentage sizing, the fields fill the whole column - but have a lower width as the leader-field in the same column.

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoGridLayoutLeadingComponent}")

public class DemoGridLayoutLeadingComponent
    extends WorkpageDispatchedPageBean 
    implements Serializable
{
    public DemoGridLayoutLeadingComponent(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demogridlayoutleadingcomponent.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoGridLayoutLeadingComponent}"; }

}
