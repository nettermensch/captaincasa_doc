The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Workplace Management
  • Immediately opening popup

The content page is opened as popup, inside the content page there is another popup that is internally opened when starting the content page.

The opening of the page is done within the workplace tree (workplace/WPFunctionTreeGeneral.java):

public class PpopupImmediateOpenFunctionNode extends FunctionNode
{
public PpopupImmediateOpenFunctionNode(FIXGRIDTreeItem parentNode, String page)
{
super(parentNode,page);
}
@Override
public void onRowExecute()
{
IWorkpage wp = loadContentPage(getWorkpageStartInfo());
DemoPopupImmediateOpen pip = (DemoPopupImmediateOpen)wp.getDispatcher().getDispatchedBean(DemoPopupImmediateOpen.class);
pip.prepare();
}
}

and:

sub = new PpopupImmediateOpenFunctionNode(top,"/workplace/demopopupimmediateopen.jsp"); sub.setOpenAsPopupByDefault(true);
sub.setText("Immediately opening popup"); sub.setStatus(FunctionNode.STATUS_ENDNODE);



There are some rules you need to pay attention to:

(.) The opening of the popup should be done within the invoke phase of the request processing. It is not a good idea to open the popup as part of the constructor processing of the page that is opened within the workplace.

Background: the constructor typically is called (when being accessed first time) within the rendering phase - at this phase you should not manipulate the current page anymore (parts of it may already be rendered!), and opening a popup means an update for the page. This can lead to "funny" situations, in which a popup is opened within the constructor of the opended workpage-bean, but becomes visible one roundtrip later.

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.ModalPopup;
import org.eclnt.jsfserver.elements.util.DefaultModalPopupListener;
import org.eclnt.jsfserver.util.ClientTestLog;
import org.eclnt.jsfserver.util.IClientLogAware;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoPopupImmediateOpen}")

public class DemoPopupImmediateOpen 
    extends PageBean
    implements Serializable
{
    ModalPopup m_popup;

    public DemoPopupImmediateOpen(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
    }

    /*
     *  The prepare is called from the workplace processing (workplace/WPFunctionTreeGeneral.java).
     */
    public void prepare()
    {
        ClientTestLog.addClientTestLogMessage("Preparing popup");
        m_popup = getOwningDispatcher().createModalPopup();
        m_popup.open("/workplace/demopopupimmediateopen2.jsp","Title of Popup",400,300,new DefaultModalPopupListener(m_popup));
        m_popup.setLeftTopReferenceCentered();
    }

    public void onCloseWorkpage(ActionEvent event)
    {
        getWorkpageContainer().closeWorkpage(getWorkpage());
    }
    
    public void onClose(ActionEvent event)
    {
        try
        {
            ClientTestLog.addClientTestLogMessage("Closing popup");
            m_popup.close();
        }
        catch (Throwable t)
        {
        }
    }

    
}
