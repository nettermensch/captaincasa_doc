The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Field Control
  • Date Input
  • Date formatting (1)

When using "date" as format you either can select various variants for formatting the date:

(.) as with all other time-related formats you can use "short", "medium", "long" - these rules are interpreted within the Java date formatting rules corresponding to standards defined by the current Locale.
(.) you can use any "free style" formatting rules corresponding to what is possible with "SimpleDateFormat". Details are available via http://java.sun.com/j2se/1.5.0/docs/api/java/text/SimpleDateFormat.html

When using "free style" formatting rules, then internationalization issues are on your side. I.e. you typically do not assign these rules directly but through a resource file - so that the formatting depends from language and/or country of the user.

This is the Java code:

package workplace;

import java.io.Serializable;
import java.util.Date;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.util.ClientTestLog;
import org.eclnt.jsfserver.util.IClientLogAware;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoCalendarFormat}")

public class DemoCalendarFormat
    extends PageBean
    implements Serializable, IClientLogAware
{
    public DemoCalendarFormat(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("calendarfield"); 
    }
    
    protected Date m_date = new Date();
    public Date getDate() { return m_date; }
    public void setDate(Date value) { m_date = value; }
    
    public void onDateAction(ActionEvent event)
    {
        ClientTestLog.addClientTestLogComment("Date was changed");
    }
    
    public boolean onCreateClientTestLogMessage()
    {
        ClientTestLog.addClientTestLogMessage("Date: " + m_date.getTime());
        return false;
    }
    
}
