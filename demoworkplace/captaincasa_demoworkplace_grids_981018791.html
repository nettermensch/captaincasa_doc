The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Grids & Trees
  • Grids and Trees
  • Selection Issues
  • Selection Roundtrips

Change data within the grids on the left and navigate through the grids (e.g. by cursor up/down): each grid will show different roundtrip behaviour.

By default each row selection inside a grid triggers a roundtrip to the server side. As consequence the server side application might update other data (e.g. the detail display of the selected item).

You can override this behaviour by using the attribute FIXGRID-AVOIDROUNDTRIPS:

When defining "1" then roundtrips are only triggered if the row that you just leave contains changed data.
When defining "2" then row selections in general do not cause any server side communication.

Typical usage scenarios: the default roundtrip behaviour is the one to be preferred when a selection of a row should take immediate effect for other parts of your layout. The "1" roundtrip behaviour is the one that is useful when maintaining data within the grid - then with leaving the grid row a corresponding communication to the sever is triggered, while normal grid navigations (up/down) do not trigger a roundtrip. The "2" is the one to be used when any type of processing is triggered from outside anyway (e.g. from a button).

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.elements.impl.FIXGRIDItem;
import org.eclnt.jsfserver.elements.impl.FIXGRIDListBinding;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoGridAvoidRoundtrips}")

public class DemoGridAvoidRoundtrips
    extends WorkpageDispatchedPageBean
    implements Serializable
{
    public class GridItem extends FIXGRIDItem implements java.io.Serializable
    {
        int m_revenue;
        public void setRevenue(int value) { m_revenue = value; }
        public int getRevenue() { return m_revenue; }
        int m_cost;
        public void setCost(int value) { m_cost = value; }
        public int getCost() { return m_cost; }
    }
    
    public class Grid extends FIXGRIDListBinding<GridItem>
    {
        public int getRevenueAll()
        {
            int result = 0;
            for (GridItem gi: getItems())
                result += gi.getRevenue();
            return result;
        }
        public int getCostAll()
        {
            int result = 0;
            for (GridItem gi: getItems())
                result += gi.getCost();
            return result;
        }
    }
    
    int m_roundtripCounter;
    protected Grid m_grid1 = new Grid();
    protected Grid m_grid2 = new Grid();
    protected Grid m_grid3 = new Grid();
    
    public DemoGridAvoidRoundtrips(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        for (int i=0; i<4; i++)
        {
            m_grid1.getItems().add(new GridItem());
            m_grid2.getItems().add(new GridItem());
            m_grid3.getItems().add(new GridItem());
        }
    }
    
    public Grid getGrid1() { return m_grid1; }
    public Grid getGrid2() { return m_grid2; }
    public Grid getGrid3() { return m_grid3; }
    
    public int getRoundtripCounter() 
    {
        m_roundtripCounter++;
        return m_roundtripCounter; 
    }

}
