The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Touch Screen
  • Touch Grid Navigation

This demo shows how to use buttons for scrolling wihtin a grid. There are two ways to go:

1. use normal buttons and link them to the server side processing
2. use special touch buttons

1. Usage of normal buttons:

On server side there are a couple of default functions available with each FIXGRIDBinding implementation. Within the property "scrollHelper" you find functions that you can simply bind to buttons/icons or other components.

There are two categories of scroll functions:
(1) the "onScroll*"-functions just scroll the list - without any consequence on selected items
(2) the "onSelect*"-functions both scroll the list and select one item

2. Usage of touch buttons

There is a special component type FIXGRIDTOUCHBUTTON. With each definition you define the id of the grid that you refer to and you assign a command associated with the button. So, this is one of the rare cases in which you really have to assign an explicit id to a component (the FIXGRID), because you reference it from other components (the FIXGRIDTOUCHBUTTONs).


In addition you see that there is a special selection mode available for the selection of multiple items: by default you need to press the ctrl- or shift-key and click with the mouse on the items that you want to select. When changing the attribute MULTISELECTMODE then items are selected/deselected by simple clicking (there is no keyboard with touch screens...)

Please also be aware that there is a client start attribute "touch" (not part of this demo). If setting "touch=true" then all scrollbars are magnified by a certain factor so that they are much simpler to use via a touch screen device.

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.elements.impl.FIXGRIDItem;
import org.eclnt.jsfserver.elements.impl.FIXGRIDListBinding;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoTouchGrid}")

public class DemoTouchGrid
    extends PageBean
    implements Serializable
{
    public class GridItem extends FIXGRIDItem implements java.io.Serializable
    {
        protected String m_articleText;
        public String getArticleText() { return m_articleText; }
        public void setArticleText(String value) { m_articleText = value; }
        
        protected String m_articleId;
        public String getArticleId() { return m_articleId; }
        public void setArticleId(String value) { m_articleId = value; }
    }
    
    protected boolean m_mutliSelection;
    protected FIXGRIDListBinding<GridItem> m_grid = new FIXGRIDListBinding<GridItem>();
    
    public DemoTouchGrid(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        for (int i=0; i<100; i++)
        {
            GridItem gi = new GridItem();
            gi.setArticleId("471"+i);
            gi.setArticleText("Text for " + gi.getArticleId());
            m_grid.getItems().add(gi);
        }
    }

    public boolean getMutliSelection() { return m_mutliSelection; }
    public void setMutliSelection(boolean value) { m_mutliSelection = value; }
    
    public FIXGRIDListBinding<GridItem> getGrid() { return m_grid; }
}
