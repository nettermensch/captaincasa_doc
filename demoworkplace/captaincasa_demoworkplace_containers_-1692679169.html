The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Tab Pane (server side)
  • Tabbed Line Container

The TABBEDLINECONTAINER components (what a nice name...!) is a tab component, that is simple to use - and powerful.

From first point of view it is very similar to the TABBEDPANE component, but there are some significant differences:

(.) Only the TAB-content is shown that is currently selected. Each selection of a tab is sent to the server side - and the corresponding content is shown.
This is quite important from scalability point of view. With a TABBEDPANE all contained tab-content is loaded to the client, also the content that is not shown at all. With a TABBEDLINECONTAINER component only the currently visible content is sent to the client side.
(.) All the painting of the tabs is nicely control-able through various BGPAINT attributes.
(.) The TABBEDLINECONTAINER internally consists out of a TABBEDLINE that is managing a content container.

The currently selected tab can be bound to a bean property by using the VALUE attribute. In addition you can bind the action listener to the component, so that you get a Select-event-notification every time the user switches to a new tab.

Please note:
(.) the BGPAINT attribute is affecting the whole component - if you only want to address the contained container component use INNERBGPAINT
(.) the BORDER and PADDING attribute are affecting the inner container component (that is placed below the tab-line on the top).
