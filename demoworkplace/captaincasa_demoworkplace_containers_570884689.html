The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Alignment in Rows

Within a row several components can be arranged - each one having a different height. In the example the TEXTAREA on the very right of each row is having the highest height, and as consequence is driving the whole row's height.

The default vertical positioning of components inside a row is "center". This means: if the component does not occupy all the height that comes from the row, then it is automatically put into the vertical center.

By using the attribute ROWALIGNMENTY you can override this behaviour.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_3" rowdistance="#{ccstylevalue['@defaultrowdistance@']}" objectbinding="#{d.demoRowalignment}">
  <t:row id="g_4" >
    <t:label id="g_5" text="Center" width="50" />
    <t:field id="g_6" width="50" />
    <t:button id="g_7" text="OK" />
    <t:tabbedpane id="g_8" height="50" width="100" >
      <t:tabbedpanetab id="g_9" text="Tab" />

    </t:tabbedpane>
    <t:textarea id="g_11" height="100" width="100%" />
  </t:row>
  <t:row id="g_12" >
    <t:label id="g_13" rowalignmenty="top" text="Top" width="50" />
    <t:field id="g_14" rowalignmenty="top" width="50" />
    <t:button id="g_15" rowalignmenty="top" text="OK" />
    <t:tabbedpane id="g_16" height="50" rowalignmenty="top" width="100" >

  </t:row>
  <t:row id="g_20" >
    <t:label id="g_21" rowalignmenty="bottom" text="Bottom" width="50" />

  </t:row>
</t:rowdemobodypane>
<t:pageaddons/>
</t:layoutdefinition>
