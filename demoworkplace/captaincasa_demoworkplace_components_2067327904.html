The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Menus
  • Button Popup

The BUTTONPOPUP component allows to open up a content area when the user presses the button. Within the content area a pane can be placed - holding any additional content.

The content area is opened within a popup menu: the popup menu is closed automatically when the user pressed the mouse outside the content area. There is no notification on this event.

In addition it is not possibile to use components within the content area that themselves open up a dialog window, e.g.: combo boxes, or right mouse button menus.

Use the BUTTONPOPUP component for "menu like" purposes. Do not use it as replacement for modal or modeless dialogs - because you do not have the programmatic control on opening and closing the area.

Please pay attention: The BUTTONPOPUP component is internally implemented as Menu. It is only possible to open one menu at one time, so you cannot use the component for opening complex screens containing menus itself, or containing any other component that internally opens up a menu-popup. This includes all COMBOBOX components (COMBOBOX, AUTOCOMPLETE). Use the normal modeless popup (with options: "undecorated", "close on click outside") to open up complex screens.

public class DemoButtonpopup extends PageBean
{
    public DemoButtonpopup(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("buttonpopup");
    }
    
    public void onAlignCenter(ActionEvent event)
    {
        OKPopup op = OKPopup.createInstance("Align Center","The icon >align,center< was pressed.");
    }
