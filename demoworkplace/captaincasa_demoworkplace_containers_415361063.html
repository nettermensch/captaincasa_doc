The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Sizeable Pane

In each foldable pane a SIZEABLEPANE component is placed: on the bottom of each you see a shaded line that you can drag and drop for changing the height of the pane.

Please note, when using the component SIZEABLEPANE: the components that are placed into the pane need to be percentage-sized, otherwise no sizing will be enabled - it does not make sense to have dynamic sizing around fix-sized content. In the example it is the foldable pane with a height of 100% that is the one to be sized.

The height is returned in pixel values, taking into consideration that there is a certain scaling on client side. By using the attribute FLUSH you can define that height updates are sent immediately after having changed the height.

The component is somehow similar with the SPLITPANE component, but: each SIZEABLEPANE is sized independent from other panes - whereas the SPLITPANE component always sizes (exactly) two panes that are contained.

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.managedbean.IDispatcher;

@CCGenClass (expressionBase="#{d.DemoSizeablepane}")

public class DemoSizeablepane 
    extends PageBean
{
    public DemoSizeablepane(IDispatcher dispatcher)
    {
        super(dispatcher);
    }

    protected String m_bottomSize = "50%;200";
    public String getBottomSize() { return m_bottomSize; }
    public void setBottomSize(String value) { m_bottomSize = value; }

    protected String m_topSize = "50%;200";
    public String getTopSize() { return m_topSize; }
    public void setTopSize(String value) { m_topSize = value; }

}
