The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Menus
  • Button Popup - Explicitly controlling animations

The dialog that is shown when pressing/activating the BUTTONPOPUP control can be animated.

As with "normal" popup dialogs there are two animation types that are applied:

. POPUPSIZENIMATIONTYPE defines the size animation - by default the animation type is resolved automatically
. POPUPANIMATIONTYPE defines the content animation - by default there is no explicit content animation

In the examples the size animation was set to "noanimation", so that the popup dialog appears with its full size immediately. As result the content animations are "fully visible". - You may of course also combine size animation and content animations.
