The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Invokers
  • Button with HTML text

By assigning a text in the format "<html>...</html>" the text is passed as HTML content into the component.

Internally the client does some HTML-sanitizing against the HTML 4.0 standard in order to avoid cross site scriping attacks. Example: the sanitizing e.g. also removes the "src" attribute of images, because you/someone may pass "javascript:" statements inside. As consequence images have to be passed by using e.g. SPAN or other HTML elements - together with some background styling.

Please note: the height-sizing of the button by default follows the font that is used within the button (FONT attribute, if not set: style). Only if the button text contains some "<br>" then the height will be calculated by analyzing the contained HTML text.

Please note: from performance point of view "<html>...</html>" buttons are slower than buttons with plain text. You should only use this feature when rendering HTML-content!
