The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Drag & Drop
  • Define explicit content that is transferred

When working with expressions then there is a useful feature: you may define the DRAGSEND definition so that it contains two types of content information:

1. the one that is used for matching the DRAGSEND info against the DROPRECEIVE info
2. the one that is the actual content that is passed when dropping

In the demo: "INFO:aaa" and "INFO:bbb" are the DRAGSEND definitions that used for matching the expression of the receiver side. When the drop is done then the "INFO.content:..." information is actually sent to the server side.

This is exactly the naming convention: if your DRAGSEND contains a content type "xxx", then the information that is sent in drop is taken from content type "xxx.content" - if available.

This is the XML layout definition:

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.elements.events.BaseActionEventDrop;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoDragContent}")

public class DemoDragContent
    extends WorkpageDispatchedPageBean 
    implements Serializable
{
    public DemoDragContent(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demodragcontent.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoDragContent}"; }

    public void onDropAction(javax.faces.event.ActionEvent event) 
    {
        if (event instanceof BaseActionEventDrop)
        {
            BaseActionEventDrop e = (BaseActionEventDrop)event;
            Statusbar.outputMessage(e.getDragInfo());
        }
    }
}
