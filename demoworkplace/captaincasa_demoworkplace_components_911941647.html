The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Check Box & Radio Button
  • Radio Button

Radio buttons are logically structured in groups. A group name is defined by the attribute GROUP - it must be unique within one page. Each group maintains one shared value, that is referenced using the VALUE attribute of each RADIOBUTTON. The REFVALUE attribute defined in the RADIOBUTTON defines the value that is represented by the corresponding button.

There is one optional feature used in the demo: the ISDEFAULT attribute. It can be set with exactly one item of the group. This item is switched on if the referenced VALUE is null, as well.

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.managedbean.IDispatcher;

@CCGenClass (expressionBase="#{d.DemoRadioButton}")

public class DemoRadioButton
    extends WorkpageDispatchedPageBean
    implements Serializable
{
    public DemoRadioButton(IDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("radiobutton");
    }

    protected String m_color;
    public String getColor() { return m_color; }
    public void setColor(String value) { m_color = value; }

    public void onColorNull(ActionEvent event) 
    {
        m_color = null;
    }

}
