The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Client Integration
  • Client HTTP Receiver (Java Client only)

Open up a browser (Firefox, Internet Explorer) and open the URL http://localhost:4444/Hello - you will see an event inside the protocol that is output.

By using the component CLIENTHTTPRECEIVER you are able to send http-get-requests to the client. The client will receive the request and notiy the server processing by calling the assigned actionListener method. The query string of the http-get-request is passed as event parameter.

What is the main purpose? You can integrate other client software in a smart way. Imagine you have a barcode scanner that needs to trigger a certain client processing. The barcode scanner software can send an http request to the client processing and by this automatically trigger some server processing.

Please pay attention: any software can now send http requests to your client - when using the CLIENTHTTPRECEIVER component. You must as consequence establish corresponding security checks on server side to check if the request really can be executed. E.g. you may pass some tokens as URL parameters by which the http-sender must authenticate itself.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:beanprocessing id="g_1" >
<t:clienthttpreceiver id="g_2" actionListener="#{d.DemoClientHttpReceiver.onClientHttpReceive}" port="4444" />
</t:beanprocessing>
<t:rowdemobodypane id="g_3" rowdistance="#{ccstylevalue['@defaultrowdistance@']}" objectbinding="#{d.DemoClientHttpReceiver}">
<t:row id="g_4" >
<t:label id="g_5" text="Protocol" />
</t:row>
<t:row id="g_6" >
<t:textarea id="g_7" enabled="false" height="100%" text="#{d.DemoClientHttpReceiver.protocol}" width="100%" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.elements.events.BaseActionEventClientHttpReceive;
import org.eclnt.jsfserver.managedbean.IDispatcher;

@CCGenClass (expressionBase="#{d.DemoClientHttpReceiver}")

public class DemoClientHttpReceiver
    extends PageBean
    implements Serializable
{
    public DemoClientHttpReceiver(IDispatcher dispatcher)
    {
        super(dispatcher);
    }

    protected String m_protocol = "...no http received yet...\r\n\r\n";
    public String getProtocol() { return m_protocol; }
    public void setProtocol(String value) { m_protocol = value; }

    public void onClientHttpReceive(ActionEvent event) 
    {
        BaseActionEventClientHttpReceive baechr = (BaseActionEventClientHttpReceive)event;
        m_protocol += baechr.getQueryString() + ", " + baechr.getHostName() + ", " + baechr.getHostAddress() + "\r\n";
    }

}
