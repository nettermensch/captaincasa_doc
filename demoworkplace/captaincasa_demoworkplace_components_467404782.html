The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Animation & Graphics & Sound
  • Paint Area
  • Line drawing
  • Line with interim points

You may define "interim points" for a line. In this case the line has some "general start and end points" which is defined via the BOUNDS attribute. And it may have any number of interim points in between that are defined via the attribute INTERIMPOINTS.

The coordinates of the INTERIMPOINTS-value are pixel values, relative to the starting point of the line.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoLineInterimPoints}" >
<t:row id="g_2" >
<t:scrollpane id="g_3" height="100%" width="100%" >
<t:row id="g_4" >
<t:paintarea id="g_5" height="100%;400" width="100%;400" >
<t:paintarealineitem id="g_6" arrowfrom="1" arrowto="6" bounds="100;100;200;200" interimpoints="50;0;50;200" invokeevent="click" />
<t:paintarealineitem id="g_7" arrowfrom="1" arrowto="1" bounds="100;100;0;100" interimpoints="-20;20;20;40;-20;60;20;80" invokeevent="click" linecolor="#800000" />
</t:paintarea>
</t:row>
</t:scrollpane>
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>
