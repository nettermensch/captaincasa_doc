The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • General Issues
  • Access style values

Both in the CSS style XML definition ("riscstyle_<styleName>.xml") and in the attribute style definition ("style.xml") you may define style values:

CSS style:
<var n="@<name>@" v="<value>"/>

XML attribute style:
<stylevalue name="<name>" value="<value>"/>

These style values can be accesse from the layout by using the expression:
"#{ccstylevalue.<styleValue>}" for simple names and/or
"#{ccstylevalue['<styleValue>']} for complex names

As consequence you can outsource certain optical information into the style management.

This is the XML layout definition:

<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoStyleValueAccess}" rowdistance="#{ccstylevalue['@defaultrowdistance@']}" >
<t:row id="g_2" >

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.util.StyleManager;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedPageBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoStyleValueAccess}")

public class DemoStyleValueAccess
    extends PageBean 
    implements Serializable
{
    public DemoStyleValueAccess(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demostylevalueaccess.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoStyleValueAccess}"; }
    
    public void onRuntimeAccessAction(javax.faces.event.ActionEvent event) 
    {
        String styleValue = StyleManager.getStyleValue("@baseColor@");
        Statusbar.outputSuccessWithPopup("The style value for \"@baseColor@\" is: " + styleValue);
    }

}
