The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Arranging Content
  • Popup Dialogs
  • Popup dialogs

Popup pages are just normal pages which are called as popups.

Opening popups is done through the methods "openModalPopup(..)" or "openModelessPopup(..)" that are available with any page bean implementation. The most important parameter to pass is an instance of the page bean that you want to open as popup.

The method returns back an instance of either "ModalPopup" or "ModelessPopup" - providing a couple of methods and properties to influence the popup's size, decoration and position.


There is a special popup handling which is included in the COMBOFIELD. The COMBOFIELD sends a special request to the server - this request can e.g. open up a popup in order to provide value help to the user. There is a preconfigured "IdTextSelection" popup which you can call using the static functions of the Java class "IdTextSelection".

    public void onDepartmentAction(ActionEvent ae)
    {
        BaseActionEvent bae = (BaseActionEvent)ae;

    public void onOpenChainedYESNOPopup(ActionEvent ae)
    {
        m_chainedPopupCounter++;
        YESNOPopup ynp = YESNOPopup.createInstance
        (
            "Iteration: " + m_chainedPopupCounter,
            "Press YES for next Yes/No popup. Or NO to abort.",
            new IYesNoListener() 
            {
                public void reactOnNo()
                {
                    Statusbar.outputMessage("No was pressed inside the popup.");

                {

                    onOpenChainedYESNOPopup(null);

        );

            ynp.getModalPopup().setOwner(getWorkpage().getModelessPopupOfWorkpage());
        else
            ynp.getModalPopup().setStartfromrootwindow(true);
        ynp.getModalPopup().setLeftTopReferenceCentered();
    }
