The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Label Control
  • Posture italic and right alignment

When using fonts with posture "italic" and when same time using "right"-alignment, then browsers will probably cut some pixels from the last character.

To avoid this you can set style parameter "_contentOverflow" to "visible". This is already done within the default style classes "risclabel_contentoverflow" and "risctextpane_overflow".

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoLabelPostureItalic}")

public class DemoLabelPostureItalic
    extends PageBean 
    implements Serializable
{
    public DemoLabelPostureItalic(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demolabelpostureitalic.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoLabelPostureItalic}"; }
}
