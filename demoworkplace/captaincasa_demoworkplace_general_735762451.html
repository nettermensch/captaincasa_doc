The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • RISC Client Integration
  • Client info

The CLIENTINFOBUTTON is a button that outputs certain client information into an alert window.

The button is useful for analyzing problems that may occur on client processing side.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:beanprocessing id="g_1" beanbinding="#{d.DemoClientInfo}" />
<t:rowdemobodypane id="g_2" objectbinding="#{d.DemoClientInfo}" >
<t:row id="g_3" >
<t:clientinfobutton id="g_4" text="Client info" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoClientInfoUI}")

public class DemoClientInfo
    extends WorkpageDispatchedPageBean 
    implements Serializable
{
    public DemoClientInfo(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/democlientinfo.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoClientInfoUI}"; }
}
