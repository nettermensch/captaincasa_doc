The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Field Control
  • Password
  • Regular Expression

You can also define regular expressions behind the PASSWORD component.

Why is this important? You may use the PASSWORD-ENCRYPTION attribute in order to directly encrypt your password on client side. When doing so, then you cannot check on server side anymore if the password was entered following certain password rules (e.g. "must contain at least one digit", "must only consist out of digits", "must be 6 characters long as minimum", ...). So, you now can do these checks on client side against a regular expression.

This regular expression is typically not used in the logon dialog, but in an administration dialog, in which you let the user specifiy his/her password.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoPasswordRegex}" >
<t:row id="g_2" >
<t:label id="g_3" text="Password" width="100" />
<t:password id="g_4" regex="[0123456789]{5}" text="#{d.DemoPasswordRegex.password}" userhint="Your password must consist out of 5 digits (e.g. &#34;37589&#34;). The regular expression that is checked is defined as &#34;[0123456789]{5}&#34;." width="100" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoPasswordRegex}")

public class DemoPasswordRegex
    extends WorkpageDispatchedPageBean
    implements Serializable
{
    public DemoPasswordRegex(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("password"); 
    }
    
    protected String m_password;
    public String getPassword() { return m_password; }
    public void setPassword(String value) { m_password = value; }
}
