The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Field Control
  • Masked input
  • Masked input

With the following attributes you can define an input mask that is automtically guiding the user through the input of complex values.

1. FIELD-INPUTMASK - this is a definition of a mask. In the mask a placeholder (default "_") is used for any character that the user may input. Other characters are fix characters of the mask.

2. FIELD-INPUTMASKPLACEHOLDER - if you do not want to use "_" as placehodler then you can define your own one here.

The value that is transferred to the application is exactly the value that you see in the field - the mask characters are part of the data.

package workplace;

import org.eclnt.workplace.IWorkpageDispatcher;

{

    String m_iban;
    String m_mac;
    String m_storageLocation;

    // ------------------------------------------------------------------------

    // ------------------------------------------------------------------------
}
