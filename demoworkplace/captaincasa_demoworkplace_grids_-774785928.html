The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Grids & Trees
  • Grids and Trees
  • Column Sequence and Widths
  • Column sizing by content

By default columns are sized independent from their content by specifying the WIDTH attribute on GRIDCOL level. You either can assign pixel value ("100"), or a percentage value ("50%"), or a mixture out of both ("100%;200" == 100% but 200 pixels as minimum).

By setting the attribute GRIDCOL-DYNAMICWIDTHSIZING the calculation of the width of the column is done according to the width of its contained content. You still can define a WIDTH, which is now interpreted as the minimum width that is guaranteed in any case.

Please be aware about the fact, that the client only knows "some" of the items that are available on server side. The automatic calculation of width is based on these items. Consequence: when scrolling the grid then the width of a column may change.

Please make sure that when switching GRIDCOL-DYNAMICWIDTHSIZING to "true", that the cell component within the GRIDCOL definition does not contain a width definition on its own.

This is the Java code:

package workplace;

import java.io.Serializable;
import java.util.Random;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.elements.impl.FIXGRIDItem;
import org.eclnt.jsfserver.elements.impl.FIXGRIDListBinding;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedBean;

@CCGenClass (expressionBase="#{d.DemoGridDynamicColSizing}")

public class DemoGridDynamicColSizing
    extends PageBean 
    implements Serializable
{
    public class GridItem extends FIXGRIDItem implements java.io.Serializable
    {
        protected String i_text1;
        protected String i_text2;
        protected String i_text3;
        protected String i_text4;
        public String getText1() { return i_text1; }
        public String getText2() { return i_text2; }
        public String getText3() { return i_text3; }
        public String getText4() { return i_text4; }
    }

    protected FIXGRIDListBinding<GridItem> m_grid = new FIXGRIDListBinding<GridItem>();

    public DemoGridDynamicColSizing(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
        for (int i=0; i<100; i++)
        {
            GridItem gi = new GridItem();
            m_grid.getItems().add(gi);
        }
        updateGridLines(1f);
    }

    public FIXGRIDListBinding<GridItem> getGrid() { return m_grid; }
    
    public void onUpdateContentAction(javax.faces.event.ActionEvent event) 
    {
        float factor = 0.25f + new Random().nextFloat() * 2;
        updateGridLines(factor);
    }

    private void updateGridLines(float factor)
    {
        for (int i=0; i<100; i++)
        {
            GridItem gi = m_grid.getItems().get(i);
            gi.i_text1 = buildString(i / 2,factor);
            gi.i_text2 = buildString((100-i)/4,factor);
            gi.i_text3 = buildString(i/3,factor);
            gi.i_text4 = buildString((100-i)/3,factor);
            m_grid.getItems().add(gi);
        }
    }
    
    private String buildString(int length, float factor)
    {
        StringBuffer sb = new StringBuffer();
        int maxi = (int)Math.round(factor * (double)length);
        for (int i=0; i<maxi; i++)
            sb.append("A");
        return sb.toString();
    }
}
