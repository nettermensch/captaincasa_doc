The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Value Representation
  • Slider

The slider allows to edit integer values.

By using the attributes MINVALUE and MAXVALUE you define the number range you can select from. If you do not specify any value, then the range from 0 to 100 is selected by default.

By defining the attributes MINORTICKSPACING and MAJORTICKSPACING you set up the scale of the slider. The ouput of textual scale information follows the definition of MAJORTICKSPACING.

By using the attribute SNAPTOPTICKS you define if the slider is bound to the values that are represented by the scale, or if also between-tick values can be selected by the user.

Finally "all the other typical attributes" are available: the VALUE that transfers the selected value to the server side, the FLUSH for indicating that value changes are sent to the server side immediately, the BGPAINT for defining the background etc. etc.

This is the Java code:

package workplace;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.demoSlider}")

public class DemoSlider extends PageBean
{
    int m_shoeSize = 41;
    public int getShoeSize() { return m_shoeSize; }
    public void setShoeSize(int value) { this.m_shoeSize = value; }

    public DemoSlider(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("slider");
    }
    
    protected int m_sliderValue = 450;
    public int getSliderValue() { return m_sliderValue; }
    public void setSliderValue(int value) { m_sliderValue = value; }
}
