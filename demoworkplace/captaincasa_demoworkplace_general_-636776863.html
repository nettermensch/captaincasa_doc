The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Workplace Management
  • Inter Workpage Events
  • Sending Page

This is the "sender" part of the workpage eventing demo. Please check the documentation of the "receiver" part for more information.

The demo runs best if you open the sender part in a window on its own, i.e. if you isolate it from the workplace window.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoWpSend}">
<t:row id="g_2" >
<t:label id="g_3" text="Create Item" width="120" />
<t:button id="g_4" actionListener="#{d.DemoWpSend.onCreateItem}" text="Execute!" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.managedbean.IDispatcher;

@CCGenClass (expressionBase="#{d.DemoWpSend}")

public class DemoWpSend 
    extends PageBean // extends WorkpageDispatcherBean
    implements Serializable
{
    public DemoWpSend(IDispatcher dispatcher)
    {
        super(dispatcher);
    }

    public void onCreateItem(ActionEvent event) 
    {
        DemoWpAddItemEvent workpageEvent = new DemoWpAddItemEvent();
        getWorkpage().throwWorkpageProcessingEvent(workpageEvent);
    }
}
