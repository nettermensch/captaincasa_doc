The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Value Representation
  • Independent scroll bar

The SCROLLBAR component is vertical scroll bar that represents/maintains an integer value. You can define the integer range (SBMINVALUE and SBMAXVALUE) and the size of the scroller (SBSIZE).

By defining the FLUSH attribute to be "true" any scroll bar changes of the user are communicated directly to the server side application processing.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoScrollBar}" rowdistance="#{ccstylevalue['@defaultrowdistance@']}" >
<t:row id="g_5" coldistance="10" >
<t:label id="g_6" text="Scrollbar" />
<t:scrollbar id="g_7" flush="true" height="250" sbmaxvalue="100" sbminvalue="0" sbsize="20" value="#{d.DemoScrollBar.verticalValue}" />
<t:coldistance id="g_2" width="20" />
<t:label id="g_13" text="Value" />
<t:field id="g_14" enabled="false" text="#{d.DemoScrollBar.verticalValue}" width="100" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoScrollBar}")

public class DemoScrollBar
    extends PageBean
    implements Serializable
{
    public DemoScrollBar(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
    }

    protected int m_verticalValue = 0;
    public int getVerticalValue() { return m_verticalValue; }
    public void setVerticalValue(int value) { m_verticalValue = value; }

}
