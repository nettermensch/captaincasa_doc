The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Scheduling
  • Schedule item variants

There are pre-configured style variants available that render the schedule item with a certain clipping area.

Check the corresponding style class definitions (e.g. "riscscheduleitem_gantt1"): in the style class there the definition of a clip-path using polygons:

clip-path: polygon(0px 0px, 100% 0px, 100% 100%, calc(100% - 5px) calc(100% - 5px), 5px calc(100% - 5px), 0px 100%, 0px 0px)

Of course you may add own style classes with own clip-path definitions.

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoScheduleItemVariants}")

public class DemoScheduleItemVariants
    extends PageBean
    implements Serializable
{
    public DemoScheduleItemVariants(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demoscheduleitemvariants.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoScheduleItemVariants}"; }
}
