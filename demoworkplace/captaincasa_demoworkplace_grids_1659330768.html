The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Grids & Trees
  • Grids and Trees
  • Large Output Lists
  • Multi label - direct usage

The MULTILABEL component renders multiple labels in one sequence. The information for the labels is contained in an object of class MULTILABELBinding.

Why is there special component to render multiple labels - and not use the normal LABEL component?
It's the speed of client side rendering. The rendering within the MULTILABEL does not render individual cells but passes all labels as one block into the DOM-tree.

The MULTILABEL component is designed to be used within the FIXGRID processing. It is implicitly used within the ARRAYGRID.

Please note:

1. You must assign background colors with a transparency (e.g. #00000010). The background is actually drawn in front of the labels in order to still see the labels' own background.
2. A selection/focus-ing of individual labels is not possible - the controls is always focused as one component.
3. On server side you can transfer the click information of an "invoke-event" so that you know which label was clicked.

Please take a look into the XML of the demo: the MULTILABEL component is not only used as content cell of the grid, but also as column header (second control instance in GRIDCOL).

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:beanprocessing id="g_1" beanbinding="#{d.DemoMultiLabel}" />
<t:rowdemobodypane id="g_2" objectbinding="#{d.DemoMultiLabel}" >
<t:row id="g_3" >
<t:fixgrid id="g_4" height="100%" objectbinding="#{d.DemoMultiLabel.grid}" selectorcolumn="2" width="100%" >
<t:gridcol id="g_5" textwidth="0" width="#{d.DemoMultiLabel.header.width}" >
<t:multilabel id="g_10" actionListener=".{onInvokeAction}" invokeevent="leftclick" multilabelbinding=".{mlb}" />
<t:multilabel id="g_11" background="#00000028" height="100%" multilabelbinding="#{d.DemoMultiLabel.header}" width="100%" />
</t:gridcol>
</t:fixgrid>
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.elements.events.BaseActionEventInvoke;
import org.eclnt.jsfserver.elements.impl.FIXGRIDItem;
import org.eclnt.jsfserver.elements.impl.FIXGRIDListBinding;
import org.eclnt.jsfserver.elements.impl.MULTILABELBinding;
import org.eclnt.jsfserver.elements.impl.MULTILABELBinding.TextInfo;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoMultiLabel}")

public class DemoMultiLabel
    extends WorkpageDispatchedPageBean 
    implements Serializable
{
    // ------------------------------------------------------------------------
    // inner classes
    // ------------------------------------------------------------------------
    
    public class GridItem extends FIXGRIDItem implements java.io.Serializable
    {    
        MULTILABELBinding m_mlb = new MULTILABELBinding();
        public MULTILABELBinding getMlb() { return m_mlb; }
        public void onRowSelect() { }
        public void onRowExecute() { }
        public void onInvokeAction(javax.faces.event.ActionEvent event) 
        {
            if (event instanceof BaseActionEventInvoke)
            {
                TextInfo ti = m_mlb.getTextInfoForEvent((BaseActionEventInvoke)event);
                if (ti != null)
                    Statusbar.outputSuccess("Event in cell: id - " + ti.getId() + ", text -" + ti.getText());
            }
        }
    }

    // ------------------------------------------------------------------------
    // members
    // ------------------------------------------------------------------------
    
    MULTILABELBinding m_header = new MULTILABELBinding();
    FIXGRIDListBinding<GridItem> m_grid = new FIXGRIDListBinding<GridItem>();
    
    // ------------------------------------------------------------------------
    // constructors & initialization
    // ------------------------------------------------------------------------

    public DemoMultiLabel(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        int maxCols = 100;
        // multi label that is shown in the gridcol header
        for (int j=0; j<maxCols; j++)
        {
            int width = (j%10)*10 + 40;
            TextInfo ti = new TextInfo().setText("Col "+j).setWidth(width);
            m_header.addTextInfo(ti);
        }
        // multi label that is shown in the grid cell per item
        for (int i=0; i<100; i++)
        {
            GridItem gi = new GridItem();
            for (int j=0;j<maxCols;j++)
            {
                int width = (j%10)*10 + 40;
                TextInfo ti = new TextInfo()
                        .setText("Test "+i+"/"+j)
                        .setWidth(width)
                        .setId(i+"/"+j);
                if (j%4 == 0)
                {
                    ti.setBackground("#FFC0C0");
                }
                if (j%7 == 0)
                {
                    ti.setBackground("#40C040");
                    ti.setForeground("#FFFFFF");
                }
                gi.getMlb().addTextInfo(ti);
            }
            m_grid.getItems().add(gi);
        }
    }

    public String getPageName() { return "/workplace/demomultilabel.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoMultiLabel}"; }

    // ------------------------------------------------------------------------
    // public usage
    // ------------------------------------------------------------------------

    public MULTILABELBinding getHeader() { return m_header; }
    public FIXGRIDListBinding<GridItem> getGrid() { return m_grid; }

}
