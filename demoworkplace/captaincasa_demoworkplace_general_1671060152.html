The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • General Issues
  • Show Client Error Screen

After pressing the button the error screen will appear that normally shows up, when no connection to the server is available.

The component SHOWCLIENTERRORSCREEN is responsible for this: by triggering its trigger it will show the client error screen.

There are not a lot of use cases, in which this component really makes sense... ;-) But there is one: you want to restart the server by an application function, so the user should directly see the error screen as response on the restart request - and not see a valid screen, in which the error screen is shown after doing some client action (e.g. pressing the button).

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:beanprocessing id="g_1" >
<t:showclienterrorscreen id="g_2" trigger="#{d.DemoShowClientErrorScreen.trigger}" />
</t:beanprocessing>
<t:rowdemobodypane id="g_3" objectbinding="#{d.DemoShowClientErrorScreen}" >
<t:row id="g_4" >
<t:button id="g_5" actionListener="#{d.DemoShowClientErrorScreen.onShowClientErrorScreen}" text="Show Client Error Screen" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.elements.util.Trigger;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedBean;

@CCGenClass (expressionBase="#{d.DemoShowClientErrorScreen}")

public class DemoShowClientErrorScreen
    extends WorkpageDispatchedPageBean 
    implements Serializable
{
    Trigger m_trigger = new Trigger();
    
    public DemoShowClientErrorScreen(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public Trigger getTrigger() 
    { 
        return m_trigger; 
    }

    public void onShowClientErrorScreen(ActionEvent event) 
    {
        m_trigger.trigger();
    }

}
