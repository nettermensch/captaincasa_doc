The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Arranging Content
  • Popup Dialogs
  • Visibility of parent hotkey definitions

By default hotkey definitions within the parent of a popup dialog are not visible inside the dialog itself. The dialog forms a "hotkey sandbox".

You can override by calling Modal/ModelessPopup.setParentHotkeysActive(true) - in this case the hotkey definitions of the parent are processed in the dialog as well.

In the example the first dialog is following the default hotkey processing - i.e. there is no visibility of parent hotkeys in the valid-values-dialog.

In the second example the parent hotkeys are visible.
Please note: due to the fact that the popup internally is defined with "closeonclickoutside", the dialog immediately closes after invoking one of the hotkeys.

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.ISetId;
import org.eclnt.jsfserver.defaultscreens.IdTextSelection;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoHotkeyAndPopup}")

public class DemoHotkeyAndPopup
    extends PageBean
    implements Serializable
{
    // ------------------------------------------------------------------------
    // members
    // ------------------------------------------------------------------------
    
    String m_department;
    
    // ------------------------------------------------------------------------
    // constructors & initialization
    // ------------------------------------------------------------------------

    public DemoHotkeyAndPopup(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
    }
    
    public String getPageName() { return "/workplace/demohotkeyandpopup.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoHotkeyAndPopup}"; }

    // ------------------------------------------------------------------------
    // public usage
    // ------------------------------------------------------------------------

    public void onDepartmentAction1(javax.faces.event.ActionEvent event) 
    {
        IdTextSelection idts = createValidValuesPopup();
    }

    public void onDepartmentAction2(javax.faces.event.ActionEvent event) 
    {
        IdTextSelection idts = createValidValuesPopup();
        idts.getPopup().setParentHotkeysActive(true);
    }

    public String getDepartment() { return m_department; }
    public void setDepartment(String value) { this.m_department = value; }

    public void onHotkey1Action(ActionEvent event)
    {
        Statusbar.outputMessage("Action 1 triggered: " + System.currentTimeMillis());
    }
    
    public void onHotkey2Action(ActionEvent event)
    {
        Statusbar.outputMessage("Action 2 triggered: " + System.currentTimeMillis());
    }
    
    // ------------------------------------------------------------------------
    // private usage
    // ------------------------------------------------------------------------
    
    private IdTextSelection createValidValuesPopup()
    {
        IdTextSelection idts = IdTextSelection.createInstance();
        idts.addLine("SALES","Sales");
        idts.addLine("PRODUCTION","Production");
        idts.addLine("QA","Quality Assurance");
        idts.addLine("FINANCE","Finance & Controlling");
        idts.setCallBack(new ISetId()
        {
            @Override
            public void setId(String id)
            {
                m_department = id;
            }
        });
        return idts;
    }

}
