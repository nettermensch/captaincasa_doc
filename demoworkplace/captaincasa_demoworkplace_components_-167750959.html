The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Check Box & Radio Button
  • Image Toggle

The TOGGLE component is very similar to a CHECKBOX - you can maintain a boolean value. For each state (true/false) you can define a specific image to be rendered. Use attributes IMAGETRUE and IMAGEFALSE. There is a default image representing a switch, that is shown if no images are specified - but we strongly recommend to use own images!

On server side the TOGGLE is represented by a boolean/Boolean value. In case of using "B"oolean, a null value is represented by a corresponding image defined by attribute IMAGENULL.

This is the Java code:

package workplace;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoToggle}")

public class DemoToggle extends PageBean
{
    public void onDayAction(ActionEvent event) 
    {
        Statusbar.outputSuccess("DAY changed: " + m_day);
    }

    public DemoToggle(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("toggle");
    }

    protected Boolean m_day = null;
    public Boolean getDay() { return m_day; }
    public void setDay(Boolean value) { m_day = value; }

    protected boolean m_married = false;
    public boolean getMarried() { return m_married; }
    public void setMarried(boolean value) { m_married = value; }
}
