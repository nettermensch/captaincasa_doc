The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Split Pane - Extended Functions
  • Write back Divider Location

Move the divider within the split pane and press "Synchronize" afterwards: the current location of the divider will be transferred back to the server.

By setting attribute WITHWRITEBACKDIVIDERLOCATION you define that user updates to the split pane update the corresponding property on server side.

As usual within the CaptainCasa client changes to the property are queued at client side, waiting for the next roundtrip. By setting FLUSH to true, the changes are immediately published - together with an action event.

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoSplitPaneSet}")

public class DemoSplitPaneSet
    extends WorkpageDispatchedPageBean 
    implements Serializable
{
    protected String m_dividerLocation = "50%";

    public DemoSplitPaneSet(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getDividerLocation() { return m_dividerLocation; }
    public void setDividerLocation(String value) { m_dividerLocation = value; }
}
