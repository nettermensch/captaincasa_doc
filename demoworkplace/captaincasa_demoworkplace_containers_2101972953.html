The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Pane with synchronized Columns

When using the default CaptainCasa containers, then each row inside the container is rendered individually. The controls within the row are NOT synchronized.

The COLSYNCHEDPANE container behaves in a different way: the width of the columns are synchronized across its contained rows. (This is very close to the behavior of nomal HTML tables...) This makes the arranging of controls simpler in certain scenarios.

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoColSynchedPane}")

public class DemoColSynchedPane
    extends PageBean 
    implements Serializable
{
    int m_fieldwidth = 100;
    public int getFieldwidth() { return m_fieldwidth; }
    public void setFieldwidth(int value) { this.m_fieldwidth = value; }

    // ------------------------------------------------------------------------
    // constructors & initialization
    // ------------------------------------------------------------------------

    public DemoColSynchedPane(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);       
        this.initRefComponent("colsynchedpane");
    }

}
