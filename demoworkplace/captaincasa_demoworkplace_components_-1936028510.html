The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Animation & Graphics & Sound
  • Paint Area
  • Line drawing
  • Lines

Lines can be easily drawn within a paint area. Pay attention: the bounds specifying the coordinates are defining a rectangle, i.e. you need to define in the way: x,y,width,height and not: x1,y1,x2,y2,

For each line you may define a text and an arrow for the starting and ending point.

Operations like selection (attribute INVOKEEVENT) and/or right mouse button menu (attribute POPUPMENU) are available as well. Events are delegated - just with every component - to the corresponding actionListener of the line.

This is the Java code:

package workplace;

import java.io.Serializable;
import java.util.Date;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.elements.events.BaseActionEventInvoke;
import org.eclnt.jsfserver.elements.events.BaseActionEventPopupMenuItem;
import org.eclnt.jsfserver.managedbean.IDispatcher;

@CCGenClass (expressionBase="#{d.DemoPaintAreaLines}")

public class DemoPaintAreaLines
    extends PageBean
    implements Serializable
{
    public DemoPaintAreaLines(IDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("paintarealineitem");
    }
    
    public void onLineAction(ActionEvent event)
    {
        if (event instanceof BaseActionEventInvoke)
        {
            Statusbar.outputSuccess("Line was clicked! Timestamp: " + (new Date()).getTime());
        }
        else if (event instanceof BaseActionEventPopupMenuItem)
        {
            BaseActionEventPopupMenuItem baepmi = (BaseActionEventPopupMenuItem)event;
            Statusbar.outputSuccess("Popup menu was executed! Command = " + baepmi.getCommand());
        }
        else
        {
            Statusbar.outputError("Hmmm: " + event.getClass());
        }
    }
}
