The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Dynamic Layout
  • Dynamic Page Generation

Normally a page name (e.g. "/workplace/demohelloworld.jsp") that is embedded within a ROWINCLUDE component points to a file that is located within you web content.

You also can resolve a page name by a program, so that the content of the page is generated dynamically.

You do so, by defining a class that implements interface IDynamicPageProvider. The class needs to be registered within the /eclntjsfserver/config/system.xml file.

system.xml:
<system>
...
<dynamicpages>
<dynamicpageprovider name="dynamicpages.DemoDynamicPageProvider"/>
</dynamicpages>
...
</system>

In our case the implementation is rather simple:
package dynamicpages;

import org.eclnt.jsfserver.dynamicpages.IDynamicPageProvider;
import org.eclnt.util.dynaccess.IDynamicIntrospectionSupported;

public class DemoDynamicPageProvider
implements IDynamicPageProvider
{
public String getPath()
{
return "/demodyn";
}
public String readDynamicPage(String pageName)
{
return
"<t:row>"+
"<t:label text='Hello world! Page name: "+pageName+"'/>"+
"</t:row>";
}
}

You see: each dynamic provider is responsible for a certain page-name-path ("/demodyn" in this case). Within this range it is invoked by the page resolution calling method readDyniamcPage(..).

This providing of dynamic pages is a very powerful way of generating dynamic content.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoDynamicPages}" rowdistance="#{ccstylevalue['@defaultrowdistance@']}" >
<t:row id="g_2" >
<t:label id="g_3" text="Include URL" width="100" />
<t:field id="g_4" text="#{d.DemoDynamicPages.pageName}" width="100%" />
</t:row>
<t:row id="g_5" >
<t:coldistance id="g_6" width="100" />
<t:button id="g_7" actionListener="#{d.DemoDynamicPages.onLoad}" text="Load" width="100" />
</t:row>
<t:row id="g_8" >
<t:pane id="g_9" border="#FF0000" height="100%" padding="1" width="100%" >
<t:rowinclude id="g_10" page="#{d.DemoDynamicPages.includedPageName}" />
</t:pane>
</t:row>
<t:row id="g_11" >
<t:label id="g_12" align="right" font="size:10" text="#{d.DemoDynamicPages.includedPageName}" width="100%" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoDynamicPages}")

public class DemoDynamicPages
    extends PageBean
    implements Serializable
{
    protected String m_pageName = "/demodyn/test";
    protected String m_includedPageName;

    public DemoDynamicPages(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
    }
    
    public String getPageName() { return m_pageName; }
    public void setPageName(String value) { m_pageName = value; }

    public String getIncludedPageName() { return m_includedPageName; }

    public void onLoad(ActionEvent event) 
    {
        m_includedPageName = m_pageName;
    }
}
