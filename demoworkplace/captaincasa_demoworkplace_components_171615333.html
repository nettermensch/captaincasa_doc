The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Touch Screen
  • Chinese keyboard

The configuration of touch keyboards now provides the possibility to also define layouts in which you have to key in some characters in order to find the character to insert.

This is the XML layout definition:

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedPageBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoChineseTouchField}")

public class DemoChineseTouchField
    extends PageBean 
    implements Serializable
{
    // ------------------------------------------------------------------------
    // members
    // ------------------------------------------------------------------------

    String m_value = "";

    // ------------------------------------------------------------------------
    // constructors & initialization
    // ------------------------------------------------------------------------

    public DemoChineseTouchField(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demochinesetouchfield.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoChineseTouchField}"; }

    // ------------------------------------------------------------------------
    // public usage
    // ------------------------------------------------------------------------

    public String getValue() { return m_value; }
    public void setValue(String value) { this.m_value = value; }

}
