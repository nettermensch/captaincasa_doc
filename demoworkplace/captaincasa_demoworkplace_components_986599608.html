The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Images
  • Image output

Image data can be output using the IMAGE or the HEXIMAGE component.

The IMAGE component receives an IMAGE attribute value that represents an access path to an image within your web application. The value either starts with "/" (being the root of your application) or is relative to the location of your page. The client will load the image via http(s) access and then cache it.

The HEXIMAGE component receives a hexadecimal string that is translated into an image on client side. The string may represent a .png , .gif or .jpg file. Use the ValueManager for transferring a byte[] array into a hexadecimal string representation.


Both type of components support the following features:

(.) explicit setting of width and height
(.) definition if the width/height ratio is kept when being resized
(.) reacting on click events (using INVOKEEVENT attribute)
(.) popup menus, hot keys, etc. as any other component

This is the Java code:

package workplace;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.util.WebResourceReader;
import org.eclnt.util.valuemgmt.ValueManager;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.demoImage}")

public class DemoImage extends PageBean
{
    public DemoImage(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("image");
    }

    public void onImageInvoke(ActionEvent event) 
    {
        Statusbar.outputSuccess("Invoke on image!");
    }

    public String getHexi() 
    { 
        byte[] bytes = WebResourceReader.readFileIntoByteArray("/images/icons/bigmails.png");
        String result = ValueManager.encodeHexString(bytes);
        return result;
    }

}
