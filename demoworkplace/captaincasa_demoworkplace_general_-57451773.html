The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Arranging Content
  • Popup Dialogs
  • Close popup with inconsistent client data

There are situations in which you do not want data resets to block the next roundtrip.

Example: any processing that cancels some operation. In this case: the current user input anyway is not processed on server side.

Pay attention: ignoring roundtrip vetos is dangerous for all other situations. When e.g. a "Save"-operation is called, then you want a data-reset to veto the next roundtrip! Otherwise the data that is reset is immediately passed into the "Save"-processing.

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.ModalPopup;
import org.eclnt.jsfserver.defaultscreens.ModalPopup.IModalPopupListener;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoIgnoreRoundtripVetos}")

public class DemoIgnoreRoundtripVetos
    extends PageBean 
    implements Serializable
{
    // ------------------------------------------------------------------------
    // members
    // ------------------------------------------------------------------------

    int m_roundtripCounter;
    String m_stringValue;
    int m_intValue;

    // ------------------------------------------------------------------------
    // constructors & initialization
    // ------------------------------------------------------------------------

    public DemoIgnoreRoundtripVetos(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demoignoreroundtripvetos.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoIgnoreRoundtripVetos}"; }

    // ------------------------------------------------------------------------
    // public usage
    // ------------------------------------------------------------------------

    public int getRoundtripCounter() { return m_roundtripCounter; }
    public void setRoundtripCounter(int value) { this.m_roundtripCounter = value; }

    public String getStringValue() { return m_stringValue; }
    public void setStringValue(String value) { this.m_stringValue = value; }

    public int getIntValue() { return m_intValue; }
    public void setIntValue(int value) { this.m_intValue = value; }

    public void onOKDefaultAction(javax.faces.event.ActionEvent event) 
    {
        m_roundtripCounter++;
    }
    
    public void onOKIgnoreAction(javax.faces.event.ActionEvent event) 
    {
        m_roundtripCounter++;
    }

    public void onPopupDefaultAction(javax.faces.event.ActionEvent event) 
    {
        openPopup();
    }
    
    public void onPopupIgnoreAction(javax.faces.event.ActionEvent event) 
    {
        ModalPopup p = openPopup();
        p.ignoreRoundtripVetosOnClose();
    }

    // ------------------------------------------------------------------------
    // private usage
    // ------------------------------------------------------------------------
    
    private ModalPopup openPopup()
    {
        final DemoIgnoreRoundtripVetosPopup ui = new DemoIgnoreRoundtripVetosPopup();
        ModalPopup result = openModalPopup(ui,"Input some incorrect data...",0,0,new IModalPopupListener()
        {
            @Override
            public void reactOnPopupClosedByUser()
            {
                closePopup(ui);
            }
        });
        return result;
    }
}
