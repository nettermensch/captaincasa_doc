The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Mobile Periphery
  • Full screen mode

The FULLSCREENBUTTON is a normal button that is not associated with some actionListener event processing on server side - but which directly request the whole screen to be opened in full screen mode.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoFullScreenButton}">
<t:row id="g_2" >
<t:fullscreenbutton id="g_3" text="Full screen!" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedPageBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoFullScreenButton}")

public class DemoFullScreenButton
    extends WorkpageDispatchedPageBean 
    implements Serializable
{
    public DemoFullScreenButton(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demofullscreenbutton.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoFullScreenButton}"; }
}
