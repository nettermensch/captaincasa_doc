The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Text Input/Output
  • HTML Editor
  • CKEditor Version 5

The attribute SIMPLEHTMLEDITOR-BRIDGEPAGE points to an HTML page that binds the processing of the HTML editor to the CaptainCasa client processing.

By using the bridge page of the example, the version 5 of the CKEditor is embedded as HTML editor.

CaptainCasa currently provides bridges to the CKEditor (version 4,5) and to the Quill editor. Writing a bridge page is simple - so there is an open integration of other editor frameworks, too. The bridge pages are part of the eclntjsfsrver.jar file.

This is the Java code:

package workplace;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoSimpleHtmlEditor}")

public class DemoSimpleHtmlEditorCK5 
    extends PageBean
{
    protected String m_htmlText;
    protected boolean m_enabled = true;

    public DemoSimpleHtmlEditorCK5(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("simplehtmleditor");
    }
    
    public String getHtmlText() { return m_htmlText; }
    public void setHtmlText(String value) { m_htmlText = value; }
    
    public boolean getEnabled() { return m_enabled; }
    public void setEnabled(boolean enabled) { m_enabled = enabled; }

    public void onLoadText(ActionEvent event) 
    {
        m_htmlText = "<html><body style='font-family:Trebuchet MS'><p>Hello world!</p></body></html>";
    }
    
    public void onSaveText(ActionEvent event)
    {
        System.out.println("==========");
        System.out.println(m_htmlText);
        System.out.println("==========");
        Statusbar.outputSuccess("Text was processed on server side.");
    }

}
