The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Touch Screen
  • Touch field

The TOUCHFIELD component allows to input a string value by using a touch field.

There is an attribute TOUCHLAYOUT by which you can reference to client side touch-keyboard layout definitions.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoTouchField}" >
<t:row id="g_2" >
<t:label id="g_3" text="Input value by TOUCHFIELD:" />
</t:row>
<t:rowdistance id="g_4" height="5" />
<t:row id="g_5" >
<t:touchfield id="g_6" height="220" value="#{d.DemoTouchField.value}" width="500" />
</t:row>
<t:rowdistance id="g_7" height="20" />
<t:row id="g_8" >
<t:label id="g_9" text="...or via field:" width="120" />
<t:field id="g_10" flush="true" text="#{d.DemoTouchField.value}" width="100" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.managedbean.IDispatcher;

@CCGenClass (expressionBase="#{d.DemoTouchField}")

public class DemoTouchField 
    extends PageBean
    implements Serializable
{
    public DemoTouchField(IDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("touchfield");
    }
    
    protected String m_value = "ABCD";
    public String getValue() { return m_value; }
    public void setValue(String value) { m_value = value; }
}
