The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Adaptive Containers - Controls
  • Adaptive grid layout

The ADAPTIVEGRIDLAYOUTPANE is a variant of the normal GRIDLAYOUTPANE - so you controls are arranged in columns.

The adaptive character comes with the attribute ADAPTIVECOLINDICES. Here you define the re-arrangements of columns dependent from different widths that are available to the control.

In the example the grid has 6 columns - due to the control definitions that are placed inside.

The definition "1000:4-2" means that the layout is updated if the available size is lower than 1000: now each line is "broken" into 4 columns for the first part and 2 columns for the second part.

You can now do multiple definitions, e.g. "300:1-1-1-1-1-1;600:2-2-2;1000:4-2". Please pay attention to define the proper order of definitions: the lowest width first, the highest width last.

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedPageBean;

@CCGenClass (expressionBase="#{d.DemoAdaptiveGridLayout}")

public class DemoAdaptiveGridLayout
    extends PageBean 
    implements Serializable
{
    public DemoAdaptiveGridLayout(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demoadaptivegridlayout.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoAdaptiveGridLayout}"; }
}
