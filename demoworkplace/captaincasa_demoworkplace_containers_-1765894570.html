The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Layering and absolute postioning
  • Overlaying of components

The OVERLAYAREA is a container holding OVERLAYAREAAREAITEM components which themselves are containers to hold any other component.

The OVERLAYAREA components are positioned by assigning an explicit x-y-width-height-z definition. Width and height are optional - if not explicitly defined then the component is sized by its minimum size.

This demo shows the usage of the property BOUNDSANCHOR that defined to which point of the item the x,y-position is applied.
