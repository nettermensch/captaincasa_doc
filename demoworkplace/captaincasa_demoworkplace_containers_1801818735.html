The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Tab Pane (server side)
  • Usage of Selection Id

by default the VALUE of the TABBEDLINE (or TABBEDLINTECONTAINER) reflects the index of the currently selected item.

But: you can also define a SELECTIONID on item level. In this case the VALUE reflects the currently selected selecition id value.

This is the XML layout definition:

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedPageBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoTabbedLineTabSelectionId}")

public class DemoTabbedLineTabSelectionId
    extends WorkpageDispatchedPageBean 
    implements Serializable
{
    String m_tlvalue = "MIDDLE";
    public String getTlvalue() { return m_tlvalue; }
    public void setTlvalue(String value) { this.m_tlvalue = value; }

    public DemoTabbedLineTabSelectionId(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demotabbedlinetabselectionid.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoTabbedLineTabSelectionId}"; }

}
