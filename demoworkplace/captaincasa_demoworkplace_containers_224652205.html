The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Cascade rendered=false

By setting the attribute CASCADERENDEREDFALSE to "true" the PANE component is NOT rendered if all of its contained children are not rendered. The "rendered=false" information is cascaded up to the level of the PANE component.

The left PANE is a normal PANE with some padding and background definition. It is always rendered - also if it does not contain any rendered content.

The right PANE is the same PANE, the only difference: attribute CASCADERENDEREDFALSE is set to "true". As consequence it completely disappears if its content is set to "rendered=false".

The same thing is demonstrated on ROW level: when switching off both label and field, then the left yellow pane is 10 pixels higher than the right yellow pane. Reason: in the PANE definition there is a ROWDISTANCE of 10 pixels. Because of the cascading in the right pane, the row is not rendered anymore. On the left side there is no cascading and the row is rendered with empty content and as consequence the ROWDISTANCE is applied.
