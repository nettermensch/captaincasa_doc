The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Value Representation
  • Slider - Text items

This demos shows some extended attributes of the SLIDER component:

(.) the ORIENTATION defines if the slider is shown horiozntally or vertically

(.) the VALUESTRINGCONVERSION defines a translation between numbers and their label representation:

The definition is done in the following way:
number:text;number:text;number:text;...

Please pay attention: no blank spaces are allowed (if they are not part of the text definition)!
Valid example: "0:Black;1:White;2:Black and white"

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoSliderExt}" rowdistance="#{ccstylevalue['@defaultrowdistance@']}" >
<t:row id="g_2" >
<t:label id="g_3" text="Own labels" width="100" />
<t:slider id="g_4" flush="true" majortickspacing="1" maxvalue="3" minortickspacing="1" minvalue="1" snaptoticks="true" value="#{d.DemoSliderExt.colorInt}" valuestringconversion="1:red;2:green;3:blue" />
</t:row>
<t:row id="g_5" >
<t:label id="g_6" text="Vertical" width="100" />
<t:slider id="g_7" flush="true" majortickspacing="1" maxvalue="3" minortickspacing="1" minvalue="1" orientation="vertical" snaptoticks="true" value="#{d.DemoSliderExt.colorInt}" valuestringconversion="1:red;2:green;3:blue" />
</t:row>
</t:rowdemobodypane>
<t:pageaddons id="g_pa"/>
</t:layoutdefinition>
