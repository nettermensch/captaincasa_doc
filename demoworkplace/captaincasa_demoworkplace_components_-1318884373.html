The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Menus
  • Button Menu

Click onto one of the buttons and a menu will show up directly below.

In the example the menu is statically built - i.e. the menu items are part of the layout definition. Of course you can - as anywhere else - use the attribute COMPONENTBINDING to build the menu dynamically within your server side application.

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.defaultscreens.ModalPopup;
import org.eclnt.jsfserver.defaultscreens.OKPopup;
import org.eclnt.jsfserver.defaultscreens.Statusbar;
import org.eclnt.jsfserver.elements.util.DefaultModalPopupListener;
import org.eclnt.jsfserver.managedbean.IDispatcher;

@CCGenClass (expressionBase="#{d.DemoButtonMenu}")

public class DemoButtonMenu 
    extends PageBean
    implements Serializable
{
    public DemoButtonMenu(IDispatcher dispatcher)
    {
        super(dispatcher);
        initRefComponent("buttonmenu");
    }

    public void onMenu1Action(ActionEvent event) 
    {
        Statusbar.outputSuccess("Menu 1 was pressed");
    }

    public void onMenu2Action(ActionEvent event) 
    {
        Statusbar.outputSuccess("Menu 2 was pressed");
    }

    public void onMenu3Action(ActionEvent event) 
    {
        Statusbar.outputSuccess("Menu 3 was pressed");
        // open popup
        ModalPopup mp = getOwningDispatcher().createModalPopup();
        mp.open("/workplace/demohelloworld.jsp","Hello world!",600,400,new DefaultModalPopupListener(mp));
    }

    public void onButtonAction(ActionEvent event)
    {
        Statusbar.outputSuccess("Button was pressed!");
    }
}
