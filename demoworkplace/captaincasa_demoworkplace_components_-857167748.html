The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Touch Screen
  • Touch Keyboard Layouts

There are pre-configured layouts for the keyboard user interface that is opened when clicking into a touch-sensitive field: "qwert" and "abc".

You can add own layouts by editing the default file /eclntjsfserver/config/touchlayouts.xml on server side. Please have a look into the template within the same directory for more details.

The client dynamically requests the server layout definition, first trying to access the file "touchlayouts_de.xml" if the client's language setting is "de". If not finding a corresponding file then the file "touchlayouts.xml" is accessed. As consequence you can build up layouts that depend from the language.
