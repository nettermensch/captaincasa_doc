The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Tab Pane
  • Tabs arranged at bottom

By default the "tabs" of a TABBEDPANE component are arranged on the top.

By using TABBEDPANE-TABPLACEMENT you can define that the tabs are arranged at the bottom.

When using the tab on the bottom then the "*_bottom" style class definitions are used. I.e. where the normal tabbed pane is using style class "risctabbedpane", the bottom-side arrangement now uses "risctabbedpane_bottom" - same with "risctabbedpane_flat" and "risctabbedpane_flat_bottom", etc.

This is the Java code:

package workplace;

import java.io.Serializable;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoTabbedPaneTabPlacement}")

public class DemoTabbedPaneTabPlacement
    extends PageBean 
    implements Serializable
{
    public DemoTabbedPaneTabPlacement(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demotabbedpanetabplacement.xml"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoTabbedPaneTabPlacement}"; }
}
