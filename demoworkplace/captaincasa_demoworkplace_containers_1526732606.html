The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Tab Pane (client side)
  • Tab Pane - Height Definition

By default and if no explicit TABPANE-HEIGHT is defined, the height of a TABPANE follows the highest height of its contained tab components.

By using attribute TABPANE-HEIGHTIFUNSELECTED you can explicitly define a height for a TABPANE if it is not selected. In the example the tabs "tab2" and "tab3" contain a quite big content - which does not influence the height of the total TABPABE component if the tabs are not selected.

This is the XML layout definition:

<?xml version="1.0" encoding="utf-8"?>
<t:layoutdefinition xmlns:t="http://www.CaptainCasa.com/controllibrary/t" xmlns:tpbc="http://www.CaptainCasa.com/controllibrary/tpbc" >
<t:rowdemobodypane id="g_1" objectbinding="#{d.DemoTabbedPaneTabHeight}" rowdistance="10" >
<t:row id="g_2" >
<t:tabbedpane id="g_3" width="100%" onetablineonly="true" >
<t:tabbedpanetab id="g_4" padding="10" text="tab1" >

</t:row>
</t:tabbedpanetab>
<t:tabbedpanetab id="g_7" heightifunselected="0" padding="10" text="tab2" >
<t:row id="g_8" >
<t:button id="g_9" height="300" text="Some Button" width="100%" />
</t:row>
</t:tabbedpanetab>
<t:tabbedpanetab id="g_10" heightifunselected="0" padding="10" text="tab3" >

<t:button id="g_12" height="100" text="Some Button" width="100%" />

</t:tabbedpane>
</t:row>
