The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Client Integration
  • Client Applet Context (Java Client only)

The component APPLETCONTEXTSHOWDOCUMENT is a quite special one: it allows to access the applet context and to open HTML documents within the browser from which the applet was started. You can both pass the HTML page and pass the name of the target frame where to show the document.

It only operates within an applet environment. If not running within an applet environment, the component will not throw any errors but just ignore commands to open up pages.

This is the XML layout definition:

<t:appletcontextshowdocument id="g_2" target="#{d.DemoAppletContext.target}" trigger="#{d.DemoAppletContext.trigger}" url="#{d.DemoAppletContext.url}" usedesktop="true" />

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.elements.util.Trigger;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoAppletContext}")

public class DemoAppletContext
    extends PageBean
    implements Serializable
{
    public DemoAppletContext(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
    }

    protected Trigger m_trigger = new Trigger();
    protected String m_url = "http://www.captaincasa.com";
    protected String m_target = "_blank";

    public String getTarget() { return m_target; }
    public void setTarget(String value) { m_target = value; }

    public Trigger getTrigger() { return m_trigger; }
    
    public String getUrl() { return m_url; }
    public void setUrl(String value) { m_url = value; }

    public void onShowDocument(ActionEvent event) 
    {
        m_trigger.trigger();
    }
}
