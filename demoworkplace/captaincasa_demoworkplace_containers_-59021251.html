The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Containers
  • Shift Container

You can slide the content part of the container by dragging and dropping it with the mouse - or by pressing the corresponding buttons on the left/right side.

The page that is currently shown is passed by the attribute VALUE. The attribute value is passed in both directions, you can both set the page index to be opened via your server side logic - and you get notified about the user changing the currently opened page.

Please note: as usual the page index is only passed immediately from the client to the server side processing if attribute FLUSH is set to "true".

This is the Java code:

package workplace;

import java.io.Serializable;
import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.workplace.IWorkpageDispatcher;
import org.eclnt.workplace.WorkpageDispatchedPageBean;

import javax.faces.event.ActionEvent;

@CCGenClass (expressionBase="#{d.DemoShiftContainer}")

public class DemoShiftContainer
    extends PageBean 
    implements Serializable
{
    String m_orientation = "horizontal";
    public String getOrientation() { return m_orientation; }
    public void setOrientation(String value) { this.m_orientation = value; }

    int m_selectedPage = 0;
    public int getSelectedPage() { return m_selectedPage; }
    public void setSelectedPage(int value) { this.m_selectedPage = value; }

    public DemoShiftContainer(IWorkpageDispatcher workpageDispatcher)
    {
        super(workpageDispatcher);        
    }

    public String getPageName() { return "/workplace/demoshiftcontainer.jsp"; }
    public String getRootExpressionUsedInPage() { return "#{d.DemoShiftContainer}"; }
}
