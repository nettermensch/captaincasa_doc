The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • General
  • Client Integration
  • Start Client Program (Java Client only)

Using the STARTPROCESS component you can launch a client side program "by commandline". You need to pass the command line string, the working directory (optional) and a trigger in order to activate the launching.

The user will get a notification before the program gets executed - in order to avoid an uncontrolled execution of client logic.

This is the Java code:

package workplace;

import java.io.Serializable;

import javax.faces.event.ActionEvent;

import org.eclnt.editor.annotations.CCGenClass;
import org.eclnt.jsfserver.elements.util.Trigger;
import org.eclnt.workplace.IWorkpageDispatcher;

@CCGenClass (expressionBase="#{d.DemoStartProcess}")

public class DemoStartProcess
    extends PageBean
    implements Serializable
{
    protected String m_arguments;
    public String getArguments() { return m_arguments; }
    public void setArguments(String value) { m_arguments = value; }

    public DemoStartProcess(IWorkpageDispatcher dispatcher)
    {
        super(dispatcher);
    }
    
    public void onStart(ActionEvent event) 
    {
        m_trigger.trigger();
    }

    protected Trigger m_trigger = new Trigger();
    public Trigger getTrigger() { return m_trigger; }
    public void setTrigger(Trigger value) { m_trigger = value; }

    protected String m_workingDirectory;
    public String getWorkingDirectory() { return m_workingDirectory; }
    public void setWorkingDirectory(String value) { m_workingDirectory = value; }

    protected String m_commandLine;
    public String getCommandLine() { return m_commandLine; }
    public void setCommandLine(String value) { m_commandLine = value; }

}
