The context of this demo is the creation of a dialog using the CaptainCasa framework for the following purpose:

  • Components
  • Straight Controls
  • Field Control
  • Additional info in field
  • Unit text - setting max width

When defining a UNITTEXT, then the text is written at the right end of the field. This automatically reduces the size of the actual text input within the field.

By default the width of the UNITTEXT is automatically determined. All the are left of the text is available for editing. If the text exceeds more than 75% of the field's width then it is automatically cut, so that there is still some space left for input.

By using attribute UNITTEXTMAXWIDTH you can explicitly define the maximum size that the unit text should take before getting cut. You may either define a (scaled) pixel value or you may define a percentage size.

Please pay attention to pass "reasonable" values only - there is no checking if e.g. the maximum width of the unit text is bigger than the width of the field.
